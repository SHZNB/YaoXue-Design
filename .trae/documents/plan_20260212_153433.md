## 问题定位
用户反馈了两个问题：
1.  **报错**: `Could not find the 'last_played_at' column`，经查是前端代码与数据库字段名不一致（`last_played_at` vs `last_accessed_at`）。
2.  **功能缺失**: “暂不支持该学科”，提示需要实现缺失学科（如生物、地理等）的实验代码。

## 解决方案
### 1. 修复字段映射错误
*   修改 `src/components/lab/GradingSystem.tsx`，将 `last_played_at` 更正为数据库实际字段 `last_accessed_at`。

### 2. 补全缺失学科实验
根据用户“每个实验点进去都是一个小游戏”的要求，为剩余学科实现交互式实验组件：
*   **生物 (BiologyLab)**: 实现“植物生长模拟器”。通过调节光照、水分、肥料参数，观察植物生长的 3D/2D 变化（发芽->长叶->开花）。
*   **地理 (GeographyLab)**: 实现“地质构造模拟器”。通过施加压力/张力，模拟断层、褶皱的形成过程。
*   **通用回退**: 对于未具体实现的学科，提供一个通用的“实验构建中”占位组件，避免直接显示“不支持”。

### 3. 完善容器映射
*   更新 `LabContainer.tsx`，引入新创建的实验组件，并根据 `subjectName` 动态渲染对应的实验。

### 4. 质量保障 (Testing)
*   **单元测试**: 为 `ChemistryLab` 和 `BiologyLab` 编写 React Testing Library 测试用例，覆盖核心交互（如点击添加试剂、调节滑块）。
*   **集成测试**: 模拟从加载实验到评分更新的完整流程。

## 实施步骤
1.  **Hotfix**: 立即修复 `GradingSystem.tsx` 的字段名报错。
2.  **组件开发**:
    *   开发 `src/experiments/BiologyLab.tsx`（植物生长）。
    *   开发 `src/experiments/GeographyLab.tsx`（地质构造）。
3.  **容器集成**: 在 `LabContainer.tsx` 中注册新组件。
4.  **测试编写**: 创建 `__tests__` 目录并编写测试用例。
5.  **交付验证**: 生成测试报告并验证所有学科入口均可正常加载。
