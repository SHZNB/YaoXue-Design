## 问题分析
用户反馈“点击实验进不去”，经查 `SubjectLab.tsx` 中的“开始实验”按钮未绑定任何事件或链接。
用户期望“每个实验点进去都是一个小游戏样式的实验”，需要构建一个沉浸式的实验运行环境，并集成之前实现的评分、报告与协同功能。

## 解决方案
### 1. 路由与导航修复
*   **注册新路由**: 在 `App.tsx` 添加 `/lab/:subjectName/:experimentId` 动态路由，指向新的实验容器页面。
*   **列表页跳转**: 改造 `SubjectLab.tsx`，将实验卡片包裹在 `Link` 中，点击后跳转至对应实验详情页。

### 2. 实验容器 (LabContainer) 设计
创建一个通用的实验运行环境 `src/pages/LabContainer.tsx`，负责：
*   **上下文管理**: 初始化 `GradingSystem`（评分）、`useCollaboration`（协同）和 `reportGenerator`（报告）。
*   **动态加载**: 根据 `experimentId` 或类型动态加载具体的实验组件（如物理/化学引擎）。
*   **UI 布局**: 顶部显示分数/倒计时，左侧显示操作步骤指引，中间为 3D/2D 游戏化实验区，右侧为协同状态与工具栏。

### 3. 小游戏化实验组件开发
*   **物理实验 (PhysicsLab)**: 将现有的 `Lab3D` 迁移并封装为 `src/experiments/PhysicsLab.tsx`，增加具体的游戏目标（如“搭建电路点亮灯泡”或“调整透镜成像”）。
*   **化学实验 (ChemistryLab)**: 新建 `src/experiments/ChemistryLab.tsx`，实现基于状态机的 2D/3D 烧杯混合交互（拖拽试剂 -> 变色/沉淀/爆炸效果）。

### 4. 数据与状态打通
*   **评分联动**: 实验组件内部触发 `logAction`，容器层的评分系统自动监听并更新分数。
*   **结束结算**: 实验完成后弹出结算模态框，展示得分并提供“生成报告”按钮。

## 实施步骤
1.  **创建容器**: 实现 `LabContainer` 骨架与通用 UI。
2.  **路由打通**: 修改 `App.tsx` 与 `SubjectLab.tsx`，确保点击能进入容器。
3.  **组件迁移与开发**:
    *   重构 `Lab3D` 为标准物理实验组件。
    *   新建化学实验 Demo 组件。
4.  **集成测试**: 验证从列表点击 -> 进入实验 -> 操作得分 -> 生成报告的全流程。

## 交付物
*   可点击跳转的实验列表。
*   沉浸式实验运行容器（含评分/报告/协同 UI）。
*   两个可交互的实验 Demo（物理+化学）。
